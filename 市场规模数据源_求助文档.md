# 「市场规模」卡片 · 数据源求助文档

> 用途：把当前进展和卡点交给 Codex / GPT 等模型继续攻。下面每个"待解决问题"都写成了可直接提问的形式。

## 一、目标

网站（烟囱的美股学习笔记，GitHub Pages + GitHub Actions 流水线）有一张「市场规模」卡片，展示全球主要股市 + 大类资产的**总市值（美元）**：

- 股票市场：美股、A股、日本、港股、韩国（口径＝各自**整个交易所**的股票总市值）
- 大类资产：黄金、白银、比特币

**硬性要求**：完全自动、尽量精确、且必须能在 **GitHub Actions（免费公有 runner，位于美国）** 的 Python 流水线里跑通。不接受"只在本地/桌面应用开着时才更新"的方案。

## 二、当前架构（已上线）

- 流水线脚本 `scripts/update_quotes.py` 每轮（交易时段约每 15–30 分钟）生成 `data/marketscale.json`，前端 `index.html` 读取渲染。
- **黄金 / 白银**：`现货价(Yahoo GC=F / SI=F) × 地上存量常数`。黄金 216,265 吨 / World Gold Council；白银 1,751,000 吨 / CPM Group（与 companiesmarketcap 同口径）。→ **实时精确，已解决。**
- **比特币**：CoinGecko `simple/price?include_market_cap=true` 实时市值。→ **已解决。**
- **五大股市**：`真实总市值锚点(cap_usd_t) × (当前大盘指数 / 锚点日指数)`。锚点存在 `config/marketscale.json`。
  - 指数：美股 ^GSPC、A股 000300.SS、日本 ^N225、港股 ^HSI、韩国 ^KS11（均走 Yahoo，GH Actions 可达）。
  - **优点**：日更、跟随大盘涨跌，不会是死数。
  - **缺点**：这是"相对跟踪"，绝对值靠人工设定的锚点；新股上市/退市带来的漂移需定期重校（约几个百分点/年）。这就是想消灭的最后一块人工。

## 三、已经排除的路（附原因，避免重复踩坑）

1. **companiesmarketcap.com `/all-countries/`**：有每日各国总市值表，但
   - 按**公司国籍**归类、且每国只收录几百家大盘股 → 港股只算出 $1.28T（真实约 $6T）、A股 $11.24T（真实约 $16.5T），亚洲市场严重低估，不能用；
   - 仅美股那行（$75.099T）准，但**该站有反爬**：浏览器能开，GitHub Actions 服务器上的 `urllib` 直连被挡（实测失败，已回退指数跟踪）。
2. **东方财富非公开接口** `push2.eastmoney.com/api/qt/ulist.np/get?...`：实测空返回，且是未公开接口、易失效，违反"不用黑箱兜底"的原则。

## 四、新线索：AKShare（开源 Python 金融数据库，pip 可装）

- **A股（有信心）**：
  - `ak.stock_sse_summary()` → 上交所股票数据总貌，含**总市值**（数据源 www.sse.com.cn）
  - `ak.stock_szse_summary()` → 深交所股票数据总貌，含**总市值**
  - 沪 + 深 总市值 = A股整体总市值，官方口径、日更、免费。
- **港股（待确认）**：AKShare 文档提到有"市场总貌"类接口含总市值，但具体函数名/字段未坐实。
- **日本 / 韩国**：AKShare 以 A股/港股/美股为主，未见 TSE/KRX 整体总市值接口。

**问题 5（美股）**：除了被反爬的 companiesmarketcap，还有没有服务器端可抓的免费日更美股整体总市值源？候选：Wilshire 5000 全市场指数（换算）、Siblis、FRED（`WILL5000INDFC` 之类）、SEC/交易所公开数据？

## 六、可接受的兜底方案（若上述都拿不到）

维持"指数跟踪"（相对精确，月内漂移通常 <2%）+ 定期（季度）重校一次绝对锚点。重校只需更新 `config/marketscale.json` 里每个市场的 `cap_usd_t` 和 `index_anchor`（同一天配对）。

## 七、相关文件

- 流水线：`scripts/update_quotes.py` → 函数 `build_market_scale()`
- 锚点配置：`config/marketscale.json`
- 输出：`data/marketscale.json`
- 前端渲染：`index.html` 中 `#scalecard` / `#scalemenu` 一段
- 工作流：`.github/workflows/update-data.yml`（`pip install` 依赖需加在"拉取行情并计算指标"步骤前）

## 八、原则（务必遵守）

- 不用会低估的口径充数（如 companiesmarketcap 的国家行）。
- 不用未公开、易失效的黑箱接口做静默兜底；拿不到就报错、记日志、回退到已声明的方法。
- 数字必须可溯源（交易所官方 / 权威机构）。

## 九、已解决（2026-07-12，桌面端 Claude 调研 + 落地）

五个问题的最终答案（实现见 `scripts/update_anchors.py` + `.github/workflows/update-anchors.yml`）：

| 市场 | 数据源 | 形态 | 更新频率 | 美国可达性 |
|---|---|---|---|---|
| 港股 | **HKMA 金管局开放 API** `api.hkma.gov.hk/public/market-data-and-statistics/monthly-statistical-bulletin/financial/capital-market-statistics` | JSON，免注册免key。字段 `eq_mkt_ttl_stock_cap`（百万港元）+ `eq_mkt_hs_index`（**当月末恒指，与市值天然配对**） | 月更 | ✅ 实测通 |
| 日本 | **JPX 官方 Excel** `jpx.co.jp/english/markets/statistics-equities/misc/tvdivq0000001w3y-att/historical-jika.xlsx`（固定网址） | 月末总市值 by 市场分部，1949至今 | 每月1日 ~13:00 JST | ✅ 实测通 |
| A股 | 上交所 `query.sse.com.cn/commonQuery.do?sqlId=COMMON_SSE_SJ_GPSJ_GPSJZM_TJSJ_L`（需 Referer）+ 深交所 `szse.cn/api/report/ShowReport/data?...CATALOGID=1803_sczm`，单位亿元 | JSON | 日更 | ⏳ 流水线实测中；不通则转 ECS（杭州）抓 |
| 韩国 | KRX 数据门户 `data.krx.co.kr/comm/bldAttendant/getJsonData.cmd`（bld=MDCSTAT00101 全指数快照，코스피/코스닥 行 MKTCAP + KOSPI收盘） | JSON POST | 日更 | ⏳ 流水线实测中 |
| 美股 | 无免费自动绝对值源（维持锚点浮动）。优化：探测 Yahoo `^FTW5000`/`^W5000`（Wilshire 全市场指数），有则替换 ^GSPC 做浮动基准，漂移大幅缩小；绝对锚由桌面端 AI 定时任务月检 | — | — | — |

架构结论：**锚点×指数浮动的架构保留**（日内实时性它已解决），新增周更校锚脚本把"人工季度校锚"变成"官方月报自动喂锚"。所有源失败均保留旧锚+写 `data/anchors_log.json` 诊断，不造数。

当年踩坑纠偏：不必死磕"GH Actions 美国 runner 直连一切"——中国源天然该由 ECS（杭州）抓；官方统计机构（金管局/交易所统计科）早就免费公开这些数，不需要去抓 companiesmarketcap 这类反爬聚合站。
