# 烟囱的美股学习笔记 · 数据源架构说明

> 最后更新：2026-07-09（盘前分钟档上线当日）
> 本文取代 2026-07-08 那份三数据源说明——**那份是错的**，错处见文末「订正记录」。

---

## 一、一句话概括

**腾讯管盘中的价格，Nasdaq 管盘前盘后的价格，Yahoo 管所有不怎么变的统计数字。**
三个各守一段，在页面上拼成一张完整的表。新浪已彻底弃用。

## 二、三个数据源的分工

| 数据源 | 负责什么 | 多快 | 怎么拿到的 |
|---|---|---|---|
| **腾讯** | A股盘中现价与当日涨跌（**实时**）<br>美股盘中 / 港股 / 日股 / 韩股 / 台股（**均延时15分钟**，见第七点五节） | **30 秒刷新**<br>（但港美日韩台的数据本身慢15分钟） | 你的浏览器直连 |
| **Nasdaq** | 美股**盘前 / 盘后**现价与当日涨跌 | **60 秒** | 阿里云 ECS 每分钟抓，写成 `live.json` |
| **Yahoo** | PE、远期PE、财报日<br>近1周 / 近1月 / 近3月 / 近半年 / YTD / 近1年<br>52周区间、历史高点<br>（以及盘前价的**兜底**） | **6 分钟**<br>（收尾后改为 15 分钟） | GitHub Actions 流水线抓，写进 `quotes.json` |

**关键分界线：会动的数据和不会动的数据，走两条完全不同的路。**

- 价格每分每秒在变 → 走轻量快车道（浏览器直连腾讯 / ECS 直连 Nasdaq），几百毫秒就回来。
- PE、52周区间这些一天变不了几次 → 走重型流水线，它要下载 386 个标的的全年 K 线，单次跑 6 分钟。

早期的错误正是**让盘前价搭上了这趟慢车**。把它挪到快车道之后，流水线反而可以放慢到 15 分钟，更稳。

## 三、前端怎么把三份数据拼起来

页面按时段自动切换来源，你不用管：

- **盘中**（北京 21:30–次日 04:00）：腾讯 30 秒实时覆盖现价与当日涨跌。
- **盘前 / 盘后**（北京 16:00–21:30 / 04:00–08:00）：读 ECS 的 `live.json`（Nasdaq，60 秒）。
  拿不到的标的，退回流水线里 Yahoo 的盘前价。
- **休市**：显示上一交易日收盘价与当日涨跌。

页面上还有两个自动推导的量，不等流水线：**市值**（＝现价 × 锚点）和**回撤**（＝现价 ÷ 历史高点），
所以它们跟着现价实时变。

## 四、怎么一眼看出这个价格是不是盘前价

**看金色「盘前」角标。**

- 有角标 → 这是盘前（或盘后）成交价。
- 无角标 → 这只票盘前还没有成交，显示的是上一交易日的收盘价与当日涨跌。

这和富途「盘前暂无成交」的口径一致。全表实测：170 只带角标的价格全部 ≠ 昨收，19 只无角标的价格全部 = 昨收，
不存在「挂着角标却显示昨收」的情况。**盘中和休市时段角标会自动消失。**

## 五、两个访问入口

- **`https://stock.ziyuanai.top`** —— 阿里云 ECS 托管，**国内直连免翻墙**，秒开。日常用这个。
- `https://chenyanchong321.github.io/us-stock-notes/` —— GitHub Pages，海外备用入口。

内容以 GitHub 仓库为准，ECS 每 2 分钟自动同步。ECS 只额外做一件事：抓盘前价。

## 六、三个自动闹钟（都在 ECS 上，无需人工干预）

1. **每分钟**：抓 Nasdaq 盘前价 → 写 `live.json`
2. **每 6 分钟**（收尾后改 15 分钟）：触发 GitHub 流水线，更新统计数据
3. **每 2 分钟**：把 GitHub 上的网站文件同步到 ECS

## 七、两道防呆守卫（吃过亏才加的）

数据源在盘前"返回昨收冒充盘前价"，是这个项目栽过两次的坑。前端因此内置了自动识别：

- **整批守卫**：若一批报价里超过 70% 与收盘价完全相同 → 判定该源不给盘前价，**整批丢弃**，退回 Yahoo。
- **逐只守卫**：某只票的报价恰好等于收盘价 → 跳过该只，**不覆盖**已取到的正确盘前价。

所以即使数据源哪天悄悄变坏，页面**不会默默显示错价**，只会降级到 Yahoo 的 6 分钟档，并在控制台报警。

## 七点五、行情授权规则（先懂这个，再谈换源）

**免费 ＝ 延时，这是交易所的规则，不是数据源的技术问题。** 换哪家国内财经网站都一样。

| 市场 | 免费行情的性质 | 依据 |
|---|---|---|
| **A股** | **免费即实时**（Level-1，3 秒一笔快照，含五档报价） | 沪深交易所 Level-1 基础行情向公众免费 |
| **港股** | **必须延迟 ≥15 分钟**（BMP 基础行情） | 港交所硬性规定；实时需按月付授权费（约 HK$130/月） |
| **美股** | 免费为延时（约 10–15 分钟） | 展示实时需向 NYSE / Nasdaq 取得牌照 |

由此推出的事实：

- 腾讯、东方财富、新浪的**免费港股行情全都是延时 15 分钟**（东方财富官网自己就写明"港股为延时行情"）。**换到东财并不能解决延时。**
- 富途能给你实时港股，是因为**它买了港交所的授权**，不是它技术更好。
- **唯一的免费实时美股来源是 Nasdaq 官方接口**（其 API 明确区分 `realtime` / `delayed` 两档）。这正是我们 `live.json` 用的那个。
- 腾讯的 **A股是实时的**（Level-1 免费实时），所以 A股 107 行不受此问题影响。

**结论**：盘中现价的正确归属是——A股 留在腾讯（本就实时）；美股 走 Nasdaq（免费实时）；港股/日股/韩股/台股 **不存在免费实时源**，只能标注"延时 15 分钟"，或付费。

> 教训：2026-07-10 之前，页面一直宣称"盘中现价 30 秒实时"。实际上除 A股外全部延时 15 分钟。
> 是用户拿富途对照宁德时代（我们 612.50 / 富途 598.00）才发现的。
> **验证方法**见 `scripts/probe-feed-lag.md`：用腾讯自己的分时数据反查实时快照对应的是哪一分钟。

## 八、已知的盲区与限制

- **美股「夜盘」**（北京白天那段场外交易）：三个免费源都没有，是富途花钱买的数据。不过那正好是你白天忙、最不需要盯盘的时候。
- **Yahoo 从阿里云 ECS 直连会被 403 拦掉**，所以 ECS 上只能用 Nasdaq。Yahoo 只能由 GitHub 流水线来抓。
- **Nasdaq 不覆盖 OTC 和指数**（如铠侠 ADR、标普500 指数），这些标的的盘前价仍靠 Yahoo 流水线，是 6 分钟档。
- **Nasdaq 在个股盘前无成交时**会退回常规时段数据（价＝昨收、涨跌幅＝昨日的），已由上面的「逐只守卫」拦下。
- **市值为估算口径**（现价 × 锚点推导），非交易所实时口径。跨市场排序时已按汇率折算成美元。

## 订正记录：2026-07-08 那份说明错在哪

那份表写着「新浪管盘前盘后的现价（分钟级）」「雅虎管不怎么变的统计数字（半小时级）」，
并说新浪是来「填补腾讯的盘前空缺」。**两处都错：**

1. **Yahoo 一直支持盘前价。** 代码里 `preMarketPrice` 是 7月7日 17:16 加进流水线的，比新浪那层（7月8日 21:47）早一天多。
   写那份说明时，Yahoo 的盘前价已经在面板上跑了一整天。把 Yahoo 说成"只管统计数字"，会让人误以为它不支持盘前。

2. **新浪根本没有盘前数据。** 2026-07-09 盘前实测：`live.json` 的 143 只美股里，**137 只的价格与涨跌幅和昨收完全一致**
   （其余 6 只仅小数取整差异）。新浪 `gb_` 接口的"当前价"字段在盘前压根不更新。

   更糟的是，它每 60 秒用这个昨收去**覆盖** Yahoo 已经取到的正确盘前价，导致盘前显示不升反降。

所以新浪那一层的真实作用从来不是"补数据"，而是想"**补频率**"（把盘前从半小时提到 1 分钟）——
这是个频率问题，不是有无问题。当时把"频率不够"表述成了"数据源不支持"，是实打实的错误。

最终 1 分钟档由 **Nasdaq** 实现，走的仍是新浪那条 ECS 快车道。**管道是对的，只是当初往里灌错了水。**
