# GitHub Pages 部署指南（一次性操作，约 15 分钟）

## 第一步：注册 GitHub 账号（已有可跳过）

1. 打开 https://github.com/signup
2. 用邮箱 chimneycyc@gmail.com 注册，记住用户名（下文用 `你的用户名` 代替）

## 第二步：创建仓库

1. 登录后点右上角 ➕ → **New repository**
2. Repository name 填：`us-stock-notes`（或任意英文名）
3. 选 **Public**（Pages 免费版需要公开仓库）
4. 点 **Create repository**

## 第三步：上传网站文件

推荐用 **GitHub Desktop**（图形界面，最简单）：

1. 下载安装 https://desktop.github.com 并登录
2. File → Clone repository → 选刚建的 `us-stock-notes`，克隆到本地
3. 把 `site-repo` 文件夹里的**全部内容**（包括 `.github` 隐藏文件夹）复制进克隆下来的文件夹
4. 回到 GitHub Desktop，会看到所有文件待提交 → 底部填个说明（如"首次上线"）→ **Commit to main** → 右上 **Push origin**

> 也可以把这一步交给我：你在 GitHub 上生成一个 Personal Access Token 给我，我在会话里直接帮你推送。

## 第四步：开启 GitHub Pages

1. 仓库页面 → **Settings** → 左侧 **Pages**
2. Source 选 **Deploy from a branch**，Branch 选 `main`，目录选 `/ (root)` → **Save**
3. 等 1-2 分钟，页面顶部会显示你的网址：
   `https://你的用户名.github.io/us-stock-notes/`

## 第五步：验证自动更新

1. 仓库页面 → **Actions** → 左侧"每日更新行情数据" → **Run workflow** 手动跑一次
2. 跑完后（约 2-3 分钟）刷新网站，左上角"数据更新"时间变成刚才的时间即成功
3. 以后每天北京时间早上 5:30（美股收盘后）自动更新，无需任何操作

## 常见问题

- **网站 404**：Pages 首次生效要等几分钟；确认 `index.html` 在仓库根目录
- **数据没更新**：看 Actions 页面有没有红叉，把报错截图给我
- **想改更新时间**：改 `.github/workflows/update-data.yml` 里的 cron 表达式
